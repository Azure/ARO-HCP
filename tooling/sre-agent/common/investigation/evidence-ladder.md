# Evidence Ladder

Prefer stronger evidence tiers over weaker ones.

1. Tier 1: direct runtime evidence from the incident window
2. Tier 2: corroborating runtime evidence from a second signal
3. Tier 3: implementation evidence from repo files or TSGs

Rules:

- runtime evidence beats code-only reasoning
- fixture context is a search aid, not proof
- if the strongest evidence is weak, keep the answer at `Leading hypothesis`
