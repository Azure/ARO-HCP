# Scope Boundaries / Non-Goals

Avoid these by default:

- style, formatting, or naming suggestions with no correctness or safety impact
- hypothetical refactors unrelated to the submitted change
- requests for tests that already exist in the PR or nearby fixtures
- broad architecture complaints not grounded in the actual diff
- comments that merely restate what the code does
- nitpicks on generated files when the source-of-truth change is correct and the generated outputs are in sync
