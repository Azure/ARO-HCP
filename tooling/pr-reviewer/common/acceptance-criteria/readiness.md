# Reviewer Readiness Criteria

The reviewer package is in a healthy baseline state when:

- the orchestrator and all primary sub-reviewers exist
- routing, invariants, evidence format, and self-check are defined
- at least a seed set of historical fixtures and lessons exists
- eval prompts exist for representative change archetypes
- update workflow, staleness, and ownership guidance exist
- a reviewer can produce a high-signal review without depending on hidden tribal knowledge
