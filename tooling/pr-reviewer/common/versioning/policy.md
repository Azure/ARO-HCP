# Reviewer Versioning Policy

- Treat this reviewer as `v1-alpha` until it has passed several real PR reviews and eval iterations.
- Bump the reviewer version whenever routing, output contract, or core triage semantics materially change.
- Record the active version in `common/versioning/version.json`.
- Keep the authoritative asset set recorded in `common/versioning/asset-inventory.json`, and keep it aligned with `MANIFEST.md`.
