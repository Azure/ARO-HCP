# Plan: Per-test must-gather query commands in Commands page

## Context

PR 4376 added infrastructure must-gather commands to the "Commands" tab but left per-test `must-gather query` commands as a TODO (commented out). The current commented-out approach was a single global command, which isn't useful. Instead, we want per-test commands using each test's own time window, presented in a compact table with copy-to-clipboard buttons — similar in spirit to the "Test Logs" table but visually smaller. Also need to reorder `--infra-cluster` to be last in query-infra commands.

## Files to Modify

1. **`ARO-HCP/test/cmd/aro-hcp-tests/custom-link-tools/options.go`** — main logic
2. **`ARO-HCP/test/cmd/aro-hcp-tests/custom-link-tools/artifacts/custom-link-tools-commands.html.tmpl`** — template
3. **Test fixtures** — regenerated by running tests with UPDATE=true

## Step 0: Write exec plan

Write full plan to `ARO-HCP/docs/exec-plans/2026-03/per-test-must-gather-commands.md`.

## Step 1: Reorder `--infra-cluster` in query-infra commands

In `options.go:getMustGatherCommands()` (lines 647-655), move `--infra-cluster` to be the last argument in both SVC and MGMT commands:

```go
// Before:
fmt.Sprintf(`hcpctl must-gather query-infra --kusto %s --region %s --infra-cluster %s --timestamp-min "%s" --timestamp-max "%s"`, ...)
// After:
fmt.Sprintf(`hcpctl must-gather query-infra --kusto %s --region %s --timestamp-min "%s" --timestamp-max "%s" --infra-cluster %s`, ...)
```

## Step 2: Remove commented-out per-test TODO code

Remove the commented-out `queryCmd` variable and the commented `CommandInfo` entry from `getMustGatherCommands()` (lines 644-660).

## Step 3: Add per-test command generation

Add a new struct and function in `options.go`:

```go
type TestCommandRow struct {
    TestName string
    Command  string
}
```

Add function `getPerTestMustGatherCommands(timingInfo map[string]TimingInfo, subscriptionID string, kusto KustoInfo) []TestCommandRow` that:
- Iterates over `timingInfo` (same data used by Test Logs)
- For each test, builds: `hcpctl must-gather query --kusto <name> --region <region> --timestamp-min "<start>" --timestamp-max "<end>" --subscription-id <id-or-empty>`
  - `--subscription-id` is always the last argument
  - If `subscriptionID` is empty, the value after `--subscription-id` is empty string (so it renders as `--subscription-id `)
- Returns slice of `TestCommandRow`

## Step 4: Wire per-test commands into `Run()`

In `options.go:Run()`:
- Call `getPerTestMustGatherCommands(timingInfo, o.SubscriptionID, o.Kusto)`
- Pass the result to the commands template alongside existing `Commands`
- Only include `TestCommands` in template data when `len(timingInfo) > 0` (matching Test Logs behavior — if no timing metadata files exist, no per-test section)

Update the template data struct to:
```go
struct {
    Commands     []CommandInfo
    TestCommands []TestCommandRow
}
```

## Step 5: Update commands HTML template

Modify `custom-link-tools-commands.html.tmpl` to add a compact per-test table after the existing infrastructure commands section. The table should:
- Only render if `.TestCommands` is non-empty
- Have a small header like `<h3>Per-test must-gather</h3>`
- Be a simple 2-column table: Test Name | Action (copy button)
- Use a small inline `<button>` with JavaScript `navigator.clipboard.writeText()` to copy the command
- Store the command in a `data-cmd` attribute on the button
- Use compact styling (smaller padding than Test Logs table) to keep it visually smaller
- Match the existing dark theme styling

## Step 6: Update test fixtures

Run tests with `UPDATE=true` to regenerate fixtures.
